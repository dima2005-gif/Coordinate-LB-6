## Теоретичні відомості
Для обчислення площі полігону, заданого координатами вершин
```math
P = \left\{ \left( x_{0}, y_{0} \right),\left( x_{1}, y_{1} \right),...\left( x_{n}, y_{n} \right)\right\}
```
використовуються різні підходи.

## Бібліотечний метод 
Бібліотека Shapely використовує оптимізовані реалізації геометричних алгоритмів на основі GEOS (C++). Оскільки робота виконується мовою JavaScript, у якості аналога використовується бібліотека JSTS, яка реалізує подібну функціональність для обчислювальної геометрії.

Результати, отримані за допомогою JSTS, використовуються як еталонні значення (ground truth) для подальшого розрахунку похибок та порівняння з реалізаціями алгоритмів (метод Гауса та метод Монте-Карло).

## Метод Гауса (Формула шнурків / Shoelace formula)
Це аналітичний метод обчислення площі простого багатокутника за координатами його вершин. Площа визначається як половина абсолютного значення суми попарних добутків координат сусідніх вершин:
```math
S = \frac{1}{2}\left| \sum_{i=1}^{n}(x_{i}y_{i+1} - x_{i+1}y_{i}) \right|
```
де остання вершина замикається з першою, тобто:
```math
(x_{n+1},y_{n+1}) = (x_{1},y_{1})
```

## Метод Монте-Карло
Iмовірнісний метод, що базується на генерації випадкових точок у відомій області (Bounding Box), що описує полігон.
- Визначаємо прямокутник, що охоплює полігон
```math
(x_{\min},  y_{\min}, x_{\max}, y_{\max})
```
- Генерується M випадкових точок рівномірно всередині цього прямокутника.
- Підраховується кількість точок K, які потрапили всередину полігону.
- Площа полігону оцінюється за формулою:
```math
S \approx S_{box} \cdot \frac{K}{M}
```
